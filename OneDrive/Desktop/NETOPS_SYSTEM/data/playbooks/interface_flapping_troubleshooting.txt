Title:
Interface Flapping Troubleshooting Guide

Purpose:
Identify and resolve interfaces repeatedly transitioning between UP and DOWN states.

Symptoms:

* Frequent connectivity interruptions
* Link state changes
* Packet retransmissions
* User disconnects

Investigation Steps:

1. Check interface status.
   Command:
   show interfaces

2. Review interface logs.
   Command:
   show logging

3. Inspect physical cabling.

4. Verify transceiver health.

5. Examine switch port statistics.

Root Cause Indicators:

* Damaged cables
* Faulty switch ports
* Defective transceivers
* Power instability

Recommended Resolution:

1. Replace damaged cables.
2. Move connection to healthy port.
3. Replace faulty transceiver.
4. Verify power supply stability.

Expected Outcome:
Stable interface operation and uninterrupted connectivity.
