Title:
OSPF Fundamentals and Troubleshooting Reference

Overview:

OSPF (Open Shortest Path First) is a link-state routing protocol used within enterprise and service provider networks to dynamically exchange routing information.

Key Concepts:

1. OSPF Neighbor
   Routers running OSPF establish neighbor relationships and exchange routing information.

Neighbor States:

* Down
* Init
* 2-Way
* ExStart
* Exchange
* Loading
* Full

A healthy OSPF neighbor relationship is typically in the FULL state.

2. OSPF Areas
   OSPF networks are divided into logical areas to improve scalability.

Common Areas:

* Area 0 (Backbone Area)
* Area 1
* Area 2

All non-backbone areas must connect to Area 0.

3. Link-State Advertisements (LSAs)
   Routers exchange LSAs to build a complete network topology database.

Frequent LSA updates may indicate instability or topology changes.

4. SPF Calculation
   OSPF uses Dijkstra's Shortest Path First algorithm to calculate optimal routes.

Excessive SPF calculations may cause high CPU utilization.

Common Issues:

Routing Loop:
A routing loop occurs when packets continuously circulate between routers due to incorrect routing information.

Symptoms:

* High CPU utilization
* Packet loss
* Increased latency
* OSPF adjacency resets

Route Redistribution Problems:
Improper redistribution between routing protocols may create routing loops and unstable routing tables.

Useful Commands:

show ip ospf neighbor

show ip ospf database

show ip route

show process cpu

Best Practices:

* Use proper route summarization.
* Carefully configure route redistribution.
* Monitor adjacency stability.
* Avoid unnecessary topology changes.

Expected Healthy Indicators:

* Neighbors remain in FULL state.
* Stable routing table.
* Low SPF recalculation count.
* Normal CPU utilization.
